# Go 原则

若要了解 Go 中一些操作的原因，有必要了解这种编程语言背后的原则：

    Go 致力于使事情变得简单，用更少的代码行执行更多操作。
    并发是首选，函数可作为轻量线程运行。
    编译和执行速度快，目标是与 C 一样快。
    Go 要求强制转换是显式的，否则会引发编译错误。
    未使用的代码不是警告，而是错误，代码将不会编译。
    有一种官方格式设置，有助于保持项目之间的一致性。
    Go 并不适用于框架，因为它更倾向于使用标准库。
    Go 确保向后兼容性。
    Go 许可证是完全开放源代码。

# Go 特性

- 闭包与匿名函数
- 错误处理
- 函数多返回值
- 自动垃圾回收
- 接口与类型
- 并发编程
- 反射
- 内置类型丰富
